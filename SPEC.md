# zennode API Specification

Canonical reference for the zennode public API. Keep this in sync with the code.

## Crate Overview

`zennode` ‚Äî self-documenting node definitions for image processing pipelines.
`zennode-derive` ‚Äî `#[derive(Node)]` and `#[derive(NodeEnum)]` proc macros.

No external dependencies beyond `core` + `alloc`. Optional features: `derive` (default), `std` (default), `serde`, `json-schema`.

## Core Traits

### `NodeDef` ‚Äî Node Type Factory

```rust
pub trait NodeDef: Send + Sync + 'static {
    fn schema(&self) -> &'static NodeSchema;
    fn create(&self, params: &ParamMap) -> Result<Box<dyn NodeInstance>, NodeError>;
    fn from_kv(&self, kv: &mut KvPairs) -> Result<Option<Box<dyn NodeInstance>>, NodeError>;
    fn create_default(&self) -> Result<Box<dyn NodeInstance>, NodeError>; // default impl
}
```

One static singleton per node type. Generated by `#[derive(Node)]` as a unit struct with a `pub static` constant.

### `NodeInstance` ‚Äî Concrete Node with Values

```rust
pub trait NodeInstance: Send + Sync + Any {
    fn schema(&self) -> &'static NodeSchema;
    fn to_params(&self) -> ParamMap;
    fn get_param(&self, name: &str) -> Option<ParamValue>;
    fn set_param(&mut self, name: &str, value: ParamValue) -> bool;
    fn as_any(&self) -> &dyn Any;
    fn as_any_mut(&mut self) -> &mut dyn Any;
    fn clone_boxed(&self) -> Box<dyn NodeInstance>;
    fn is_identity(&self) -> bool; // default: false
}
```

| `description` | `&'static str` | One-line tooltip. From doc comments. |

### `ParamValue` (runtime)

`F32(f32)`, `I32(i32)`, `U32(u32)`, `Bool(bool)`, `Str(String)`, `Enum(String)`, `F32Array(Vec<f32>)`, `Color([f32; 4])`

Accessor methods: `as_f32()`, `as_i32()`, `as_u32()`, `as_bool()`, `as_str()`, `as_f32_array()`, `as_color()`. Cross-type coercion where safe (e.g., `I32` ‚Üí `f32`).

### Enums

**`NodeGroup`**: Decode, Encode, Tone, ToneRange, ToneMap, Color, Detail, Effects, Geometry, Layout, Canvas, Composite, Quantize, Analysis, Hdr, Raw, Auto, Other.

**`NodeRole`**: Decode, Geometry, Orient, Resize, Filter, Composite, Analysis, Quantize, Encode.

- `NodeRole::is_geometry()` returns true for `Geometry`, `Orient`, `Resize`.
- `Orient` and `Resize` are sub-roles of `Geometry` ‚Äî the bridge treats them identically but downstream crates can tag their nodes specifically.

**`SliderMapping`**: Linear, SquareFromSlider, FactorCentered, Logarithmic, NotSlider.

**`PixelFormatPreference`**: Any, OklabF32, LinearF32, PremulLinearF32, Srgb8, SceneLinearF32.

**`AlphaHandling`**: Process, Skip, RequirePremul, ModifyAlpha.

### `FormatHint`

`preferred: PixelFormatPreference`, `alpha: AlphaHandling`, `changes_dimensions: bool`, `is_neighborhood: bool`.

### `CoalesceInfo`

`group: &'static str`, `fusable: bool`, `is_target: bool`.

Adjacent nodes in the same coalesce group are fused into a single operation. The fused result is always mathematically equivalent to sequential execution. The bridge never reorders ‚Äî it only fuses adjacent compatible nodes.

## Ordering Model

**zennode does NOT reorder user-specified node sequences.** Nodes execute in the order the user declared them (RIAPI querystring position, JSON array index, or DAG edges).

`NodeRole` is a **type tag** that tells the pipeline bridge what kind of planner a node feeds into. The bridge walks the user's node list left-to-right, accumulating runs of compatible nodes. When the role changes, it flushes the accumulated run to its planner and starts a new run.

```text
User order: [orient, crop, resize, exposure, contrast, sharpen, encode]

Bridge walks left-to-right:
  orient   ‚Üí geometry run (start)
  crop     ‚Üí geometry run (extend)
  resize   ‚Üí geometry run (extend)
  exposure ‚Üí FLUSH geometry ‚Üí LayoutPlan. Filter run (start).
  contrast ‚Üí filter run (extend ‚Äî same coalesce group)
  sharpen  ‚Üí filter run (extend ‚Äî neighborhood, handled by Pipeline)
  encode   ‚Üí FLUSH filter ‚Üí FilterPipeline. Collect encode config.

Result: Source ‚Üí LayoutPlan ‚Üí FilterPipeline ‚Üí Encode
```

If the user writes `[resize, sharpen, crop]`, they get `Source ‚Üí LayoutPlan(resize) ‚Üí FilterPipeline(sharpen) ‚Üí Crop`. Different graph, different results, both correct.

## RIAPI Key-Value System

### `KvPairs`

Parse RIAPI querystrings with consumed-key tracking.

- `from_querystring(qs: &str)` ‚Äî parse URL querystring
- `take(key, consumer) -> Option<&str>` ‚Äî consume a key
- `take_f32/i32/u32/bool(key, consumer)` ‚Äî consume + parse
- `take_owned(key, consumer) -> Option<String>` ‚Äî consume, return owned
- `peek(key) -> Option<&str>` ‚Äî look without consuming
- `unconsumed()` ‚Äî iterate unconsumed keys
- `warnings()` ‚Äî accumulated warnings
- `warn(key, kind, message)` ‚Äî add a warning

Keys are case-insensitive. Duplicate keys: last wins + DuplicateKey warning.

### `KvWarningKind`

UnrecognizedKey, InvalidValue, DeprecatedKey, DuplicateKey.

## Registry

### `NodeRegistry`

- `register(def)` / `register_all(defs)` ‚Äî add node definitions
- `get(id) -> Option<&dyn NodeDef>` ‚Äî lookup by ID
- `all()` ‚Äî all registered defs
- `by_group(group)` / `by_tag(tag)` ‚Äî filtered views
- `create(id, params)` ‚Äî create instance by ID
- `from_querystring(qs)` ‚Äî parse RIAPI across all nodes ‚Üí `KvResult`

### `KvResult`

`instances: Vec<Box<dyn NodeInstance>>`, `warnings: Vec<KvWarning>`.

## Derive Macros

### `#[derive(Node)]` ‚Äî Struct Attributes

**Required:**
```
#[node(id = "crate.name")]      Permanent FQN
#[node(group = Tone)]           NodeGroup variant
#[node(role = Filter)]          NodeRole variant
```

**Optional:**
```
#[node(label = "...")]          Human label (default: struct name ‚Üí "Struct Name")
#[node(version = 2)]            Schema version (default: 1)
#[node(compat_version = 1)]     Min compat version (default: 1)
#[node(coalesce = "group")]     Coalesce group (implies fusable)
#[node(fusable)]                Can be fused in coalesce group
#[node(coalesce_target)]        This IS the fused op
#[node(neighborhood)]           Neighborhood operation
#[node(changes_dimensions)]     Changes image dimensions
#[node(format(preferred = OklabF32, alpha = Skip))]
#[node(tags("basic", "tone"))]
```

Legacy: `#[node(phase = Filter)]` is accepted as an alias for `role`.

### `#[derive(Node)]` ‚Äî Field Attributes

**`#[param(...)]`:**
```
range(-5.0..=5.0)              Min..=max
default = 0.0                  Default value
identity = 0.0                 No-op value
step = 0.1                     UI step
unit = "stops"                 Display unit
section = "Main"               UI sub-section
slider = SquareFromSlider      Slider mapping
label = "..."                  Human label (default: field name titlecased)
since = 2                      Version that added this param
labels("R", "O", "Y")         Per-element labels for [f32; N] arrays
visible_when = "mode=advanced" Conditional visibility
color                          Mark [f32; 4] as Color kind
```

**`#[kv("key1", "key2")]`:** RIAPI querystring key aliases.

### `#[derive(NodeEnum)]` ‚Äî Enum Attributes

On variants:
```
#[variant(label = "...")]       Override label
#[variant(alias = "old_name")]  Additional parse alias
```

Doc comments become variant descriptions. Names convert to snake_case.

**Generates:** `zennode_variants() -> &[EnumVariant]`, `zennode_name() -> &str`, `Display`, `FromStr`.

### Generated Items from `#[derive(Node)]`

For struct `Foo`:
1. `static __ZENODE_FOO_SCHEMA: NodeSchema` ‚Äî full schema
2. `pub struct FooNodeDef;` ‚Äî factory unit struct
3. `pub static FOO_NODE: FooNodeDef;` ‚Äî singleton
4. `impl NodeDef for FooNodeDef` ‚Äî create, from_kv, schema
5. `impl NodeInstance for Foo` ‚Äî get/set params, to_params, identity, clone, downcast

### Field Type ‚Üí ParamKind

| Rust Type | ParamKind | Required attrs |
|-----------|-----------|----------------|
| `f32` | `Float` | `range`, `default` |
| `i32` | `Int` | `range`, `default` |
| `u32` | `U32` | `range`, `default` |
| `bool` | `Bool` | `default` |
| `String` | `Str` | `default` |
| `[f32; N]` | `FloatArray` | `range`, `default`, `labels` |
| `[f32; 4]` + `color` | `Color` | `default` |

## Built-in Nodes

### `zennode.decode` (in zennode)

Format-independent decode configuration:
- `io_id` (i32) ‚Äî I/O slot identifier
- `hdr_mode` (String) ‚Äî "sdr_only" (default), "hdr_reconstruct", "preserve"
- `color_intent` (String) ‚Äî "preserve" (default), "srgb"
- `min_size` (u32) ‚Äî JPEG prescale hint (0 = none)

### `zencodecs.quality_intent` (in zencodecs, feature `zennode`)

Format selection and quality profile. QualityIntent moved from zennode to zencodecs because it's about codec selection (zencodecs' domain). Maps to imageflow's `EncoderPreset::Auto`.

- `profile` (String) ‚Äî "lowest"‚Üí"lossless" or 0-100. KV: `qp`
- `format` (String) ‚Äî "" (auto), "jpeg", "webp", etc. KV: `format`
- `dpr` (f32) ‚Äî device pixel ratio. KV: `qp.dpr`, `dpr`
- `lossless` (String) ‚Äî ""/"true"/"false"/"keep". KV: `lossless`
- `allow_webp/avif/jxl/color_profiles` (bool) ‚Äî format allowlist. KV: `accept.*`

Has `to_codec_intent()` ‚Üí `CodecIntent` for resolution via zencodecs' selection engine.

## Versioning Rules (NEVER BREAK)

1. Node IDs are permanent. Never rename or remove.
2. Params are additive only. New params: `since = N`, identity default.
3. Param semantics never change. New behavior ‚Üí new node ID.
4. KV keys are permanent. Add aliases, never remove.
5. Enum variants are permanent. Never remove.
6. Defaults are frozen per version.
7. Enums serialize as strings, never integers.

## Features

| Feature | Default | Description |
|---------|---------|-------------|
| `derive` | Yes | `#[derive(Node)]` and `#[derive(NodeEnum)]` |
| `std` | Yes | `std::error::Error` impl |
| `serde` | No | Serialize/Deserialize for param types, schema export |

## Serde Behavior (feature `serde`)

- `ParamValue` serializes untagged (f32 ‚Üí number, bool ‚Üí boolean, etc.)
- Schema types (`NodeSchema`, `ParamDesc`, etc.) implement `Serialize` only
- Enums serialize as snake_case strings
- `x-zennode-*` JSON Schema extensions for slider mappings, units, identity values

## Error Types

`NodeError`: UnknownNode, UnknownParam, TypeMismatch, OutOfRange, MissingParam, InvalidEnumVariant, Other. Implements `Display`. Implements `std::error::Error` with `std` feature.

## Pipeline Integration (zenpipe)

### Bridge: `compile_nodes()`

Converts `Vec<Box<dyn NodeInstance>>` ‚Üí `PipelineGraph` + `DecodeConfig` + `EncodeConfig`.

- Walks user node list left-to-right (no reordering)
- Adjacent geometry nodes ‚Üí fused `LayoutPlan` via `zenlayout::Pipeline`
- Adjacent filter nodes ‚Üí fused `zenfilters::Pipeline`
- Encode/decode nodes separated into config structs
- `NodeConverter` trait for extensible node‚ÜíNodeOp mapping

### Sidecar Streams

Gain maps and depth maps track through geometry ops as parallel mini-pipelines:

- `SidecarPlan::derive()` ‚Äî uses `zenlayout::IdealLayout::derive_secondary()`
- `SidecarPlan::compile()` ‚Äî builds mini `PipelineGraph` with proportional transforms
- `ProcessedSidecar` ‚Äî materialized sidecar pixels + metadata for re-embedding
- Identity detection: no transforms needed ‚Üí passthrough (zero cost)
